# Лабораторна робота №3
## Організація параллельної обробки запитів на сервері за допомогою мультиплексування

### Завдання
Реалізувати "паралельне" обслуговування кількох клієнтів за допомогою мультиплексування (select/poll).

### Реалізація

#### Мультиплексування з select()
- Сервер працює в **одному потоці**
- Одночасна обробка **до 100 клієнтів**
- Чергування між: прийняттям з'єднань, читанням даних, відправкою відповідей

#### Архітектура сервера

```
┌─────────────────────────────────────────┐
│         MultiplexedServer               │
│  ┌─────────────────────────────────┐    │
│  │         Main Loop               │    │
│  │  ┌──────────────────────────┐   │    │
│  │  │    select(read_fds)      │   │    │
│  │  └──────────┬───────────────┘   │    │
│  │             │                    │    │
│  │    ┌────────┼────────┐          │    │
│  │    ▼        ▼        ▼          │    │
│  │  Accept  Read CMD  Transfer     │    │
│  │    │        │        │          │    │
│  │    └────────┴────────┘          │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

#### Клас Client
Кожен клієнт має:
- `sock` - сокет з'єднання
- `buffer` - буфер вхідних даних
- `state` - стан (IDLE, RECEIVING_CMD, TRANSFERRING)
- `transfer_data` - дані передачі файлу
- `last_activity` - час останньої активності

### Команди

| Команда | Опис |
|---------|------|
| `ECHO <text>` | Повертає текст клієнту |
| `TIME` | Повертає час сервера |
| `UPLOAD <file>` | Завантажити файл на сервер |
| `DOWNLOAD <file>` | Скачати файл з сервера |
| `CLOSE/EXIT/QUIT` | Закрити з'єднання |

### Продуктивність

#### Час відгуку
- **target**: ping × 10
- **реалізація**: RESPONSE_TIMEOUT = 0.1с (100ms)
- Для локальної мережі (ping < 1ms): відгук < 10ms

#### Максимальна кількість клієнтів
- Обмежено `FD_SETSIZE` (зазвичай 1024 на Linux, 64 на Windows select)
- Реалізовано: MAX_CLIENTS = 100

### Файли
- `multiplex_server.py` - сервер з мультиплексуванням

### Запуск

```bash
python multiplex_server.py [port]
# Default port: 8080
```

### Тестування

#### 1. Підключення кількох клієнтів:
```bash
# Термінал 1
python ../1lab/client.py 127.0.0.1 8080

# Термінал 2
python ../1lab/client.py 127.0.0.1 8080

# Термінал 3 (telnet)
telnet 127.0.0.1 8080
```

#### 2. Одночасна передача файлів:
```bash
# Клієнт 1 - UPLOAD великого файлу
> UPLOAD large_file.bin

# Клієнт 2 - ECHO команди (не блокується)
> ECHO test
> TIME
```

### Переваги мультиплексування

1. **Економія ресурсів**
   - Один потік замість потоку на клієнта
   - Менше перемикань контексту
   - Менше споживання пам'яті

2. **Масштабованість**
   - Можливість обслуговувати тисячі клієнтів
   - Лінійне збільшення продуктивності

3. **Відсутність race conditions**
   - Немає спільних змінних між потоками
   - Простіша відладка

### Недоліки

1. **Блокування на довгих операціях**
   - Рішення: неблокуючий I/O + буферизація

2. **Не використовує багатоядерність**
   - Рішення: комбінація з multiprocessing

### select() vs poll() vs epoll()

| Метод | Linux | Windows | Масштабованість |
|-------|-------|---------|-----------------|
| select() | ✓ | ✓ | До 1024 FD |
| poll() | ✓ | ✗ | До 10000 FD |
| epoll() | ✓ | ✗ | До 1M+ FD |
| IOCP | ✗ | ✓ | До 1M+ FD |

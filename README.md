## Лабораторні роботи

### №1 - Знайомство з програмуванням сокетів (TCP)

**Протокол:** TCP (надійне з'єднання)

**Функціонал:**
- Команди: ECHO, TIME, CLOSE/EXIT/QUIT
- Передача файлів: UPLOAD/DOWNLOAD
- Відновлення після обриву з'єднання
- SO_KEEPALIVE для моніторингу
- Внеполосні дані (прогрес передачі)
- Відображення бітрейту

**Файли:**
- `1lab/server.py` - TCP сервер
- `1lab/client.py` - TCP клієнт

```bash
# Запуск сервера
python 1lab/server.py 8080

# Запуск клієнта
python 1lab/client.py 127.0.0.1 8080
```

---

### №2 - Клієнт-серверна програма для передачі файлів по UDP

**Протокол:** UDP (ненадійний, з власною надійністю)

**Реалізовані механізми:**
- ✅ Підтвердження передачі пакетів (ACK)
- ✅ Повторна передача при втраті (Retransmission)
- ✅ Ковзне вікно (Sliding Window, 10 пакетів)
- ✅ Контрольна сума (CRC32/MD5)
- ✅ Визначення втрати пакетів

**Оптимальний розмір буфера:** 1472 байти
- MTU Ethernet: 1500 байт
- IP header: 20 байт
- UDP header: 8 байт
- **Дані: 1472 байти**

**Порівняння продуктивності:**
| Протокол | Швидкість |
|----------|-----------|
| TCP | ~100 Kbps |
| UDP | ~150-200 Kbps (в 1.5-2 рази швидше) |

**Файли:**
- `2lab/udp_transfer.py` - UDP передача файлів

```bash
# Сервер (приймання файлу)
python 2lab/udp_transfer.py server 8080

# Клієнт (відправка файлу)
python 2lab/udp_transfer.py client 127.0.0.1 8080
```

### №3 - Організація параллельної обробки запитів (Мультиплексування)

**Технологія:** select() / poll()

**Функціонал:**
- Однопоточний сервер
- Паралельна обробка до 100 клієнтів
- Неблокуючий I/O
- Чергування між: accept, read, write, transfer
- Час відгуку: < ping × 10 (50ms)
- Розмір порції: 1024 байти

**Архітектура:**
```
┌─────────────────────────────────┐
│  Main Loop with select()        │
│    ├─ Accept new connections    │
│    ├─ Read commands             │
│    ├─ Process file transfers    │
│    └─ Cleanup inactive clients  │
└─────────────────────────────────┘
```

**Файли:**
- `3lab/multiplex_server.py` - Мультиплексований сервер

```bash
# Запуск сервера
python 3lab/multiplex_server.py 8080

# Підключення кількох клієнтів
python 1lab/client.py 127.0.0.1 8080  # термінал 1
python 1lab/client.py 127.0.0.1 8080  # термінал 2
telnet 127.0.0.1 8080                  # термінал 3
```

### №4 - Організація параллельної обробки запитів (Багатопроцесовість)

**Варіант:** 1 (TCP, Процеси по запиту)

**Архітектура:**
- Головний процес: accept() з'єднань
- Дочірній процес: обробка клієнта
- Динамічний пул: Nmin=2, Nmax=10
- SharedStats: спільна статистика

**Функціонал:**
- Паралельна обробка клієнтів
- Ізоляція помилок (crash одного не впливає)
- Використання багатоядерності
- Команди: ECHO, TIME, UPLOAD, DOWNLOAD, STATS, CLOSE

**Порівняння з Л.р. №3:**
| Характеристика | select() | Процеси |
|---------------|----------|---------|
| Потоки | 1 | N |
| Пам'ять | Мін. | N × 4MB |
| Ізоляція | Ні | Так |
| Стабільність | Crash=всі | Crash=один |

**Файли:**
- `4lab/multiprocess_server.py` - Багатопроцесовий сервер
- `1lab/client.py` - Клієнт (той самий)

```bash
# Запуск сервера
python 4lab/multiprocess_server.py 8080

# Клієнти (кілька терміналів)
python 1lab/client.py 127.0.0.1 8080
```

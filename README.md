## Лабораторные работы

### №1 - Знакомство с программированием сокетов (TCP)

**Протокол:** TCP (надёжное соединение)

**Функционал:**
- Команды: ECHO, TIME, CLOSE/EXIT/QUIT
- Передача файлов: UPLOAD/DOWNLOAD
- Восстановление после обрыва соединения
- SO_KEEPALIVE для мониторинга
- Внеполосные данные (прогресс передачи)
- Отображение битрейта

**Файлы:**
- `1lab/server.py` - TCP сервер
- `1lab/client.py` - TCP клиент

```bash
# Запуск сервера
python 1lab/server.py 8080

# Запуск клиента
python 1lab/client.py 127.0.0.1 8080
```

---

### №2 - Клиент-серверная программа для передачи файлов по UDP

**Протокол:** UDP (ненадёжный, с собственной надёжностью)

**Реализованные механизмы:**
- ✅ Подтверждение передачи пакетов (ACK)
- ✅ Повторная передача при потере (Retransmission)
- ✅ Скользящее окно (Sliding Window, 10 пакетов)
- ✅ Контрольная сумма (CRC32/MD5)
- ✅ Определение потери пакетов

**Оптимальный размер буфера:** 1472 байта
- MTU Ethernet: 1500 байт
- IP header: 20 байт
- UDP header: 8 байт
- **Данные: 1472 байта**

**Сравнение производительности:**
| Протокол | Скорость |
|----------|----------|
| TCP | ~100 Kbps |
| UDP | ~150-200 Kbps (в 1.5-2 раза быстрее) |

**Файлы:**
- `2lab/udp_transfer.py` - UDP передача файлов

```bash
# Сервер (приём файла)
python 2lab/udp_transfer.py server 8080

# Клиент (отправка файла)
python 2lab/udp_transfer.py client 127.0.0.1 8080
```

### №3 - Организация параллельной обработки запросов (Мультиплексирование)

**Технология:** select() / poll()

**Функционал:**
- Однопоточный сервер
- Параллельная обработка до 100 клиентов
- Неблокирующий I/O
- Черёдование между: accept, read, write, transfer
- Время отклика: < ping × 10 (50ms)
- Размер порции: 1024 байта

**Архитектура:**
```
┌─────────────────────────────────┐
│  Main Loop with select()        │
│    ├─ Accept new connections    │
│    ├─ Read commands             │
│    ├─ Process file transfers    │
│    └─ Cleanup inactive clients  │
└─────────────────────────────────┘
```

**Файлы:**
- `3lab/multiplex_server.py` - Мультиплексированный сервер

```bash
# Запуск сервера
python 3lab/multiplex_server.py 8080

# Подключение нескольких клиентов
python 1lab/client.py 127.0.0.1 8080  # терминал 1
python 1lab/client.py 127.0.0.1 8080  # терминал 2
telnet 127.0.0.1 8080                  # терминал 3
```

### №4 - Организация параллельной обработки запросов (Многопроцессность)

**Вариант:** 1 (TCP, Процессы по запросу)

**Архитектура:**
- Главный процесс: accept() соединений
- Дочерний процесс: обработка клиента
- Динамический пул: Nmin=2, Nmax=10
- SharedStats: общая статистика

**Функционал:**
- Параллельная обработка клиентов
- Изоляция ошибок (crash одного не влияет)
- Использование многоядерности
- Команды: ECHO, TIME, UPLOAD, DOWNLOAD, STATS, CLOSE

**Сравнение с Л.р. №3:**
| Характеристика | select() | Процессы |
|---------------|----------|----------|
| Потоки | 1 | N |
| Память | Мин. | N × 4MB |
| Изоляция | Нет | Да |
| Стабильность | Crash=все | Crash=один |

**Файлы:**
- `4lab/multiprocess_server.py` - Многопроцессный сервер
- `1lab/client.py` - Клиент (тот же)

```bash
# Запуск сервера
python 4lab/multiprocess_server.py 8080

# Клиенты (несколько терминалов)
python 1lab/client.py 127.0.0.1 8080
```

## Тестирование сети

### Проверка открытых портов
```bash
# Windows
netstat -an | findstr :8080

# Linux/macOS
netstat -tlnp | grep 8080
# или
ss -tlnp | grep 8080
```

### Сканирование портов (nmap)
```bash
nmap -p 1-10000 <server_ip>
```

### Измерение ping
```bash
ping 127.0.0.1
ping <server_ip>
```

### Проверка процессов (Л.р. №4)
```bash
# Windows
tasklist | findstr python

# Linux
ps aux | grep python
```
